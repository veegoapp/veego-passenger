بص فى الانترفيس بتاع الشاتيل فى صفحة الروتس مفروض عاوز اضيف حاجه :
الراكب لما يبحث عن اى روت و يدوس عليه و تقتح صفحة الروت ديتلز عاوزك تبنى زرار من فوق كده اسمه Request a Trip ده لما بيدوس عليه بتتفح صفحة فيها مواعيد ذهاب و مواعيد عوده هو بيقدر يعلم على الى هو عاوزه و يدوس اوك مفروض الداتا دى بتجيلى على الباك اند فا عاوزك تبنيلى ده و بص ده رح الباك اند للى بنيته فى الباك اند ابنى ده بناء على كلام الباك اند 
كل حاجة بُنيت بنجاح وبدون أي errors! الـ DB schema اتحدّث تلقائياً وأضاف عمود requests_enabled على جدول الـ routes.

✅ ملخص التعديلات الكاملة
🗃️ Database
•	عمود جديد requests_enabled (boolean, default false) على جدول routes
•	كل روت يتحكم في نفسه مستقل
🔌 Backend APIs الجديدة
Admin APIs (تحتاج Admin Token):
Method	Endpoint	الوصف
GET	/admin/trip-requests/routes-status	كل الروتس مع حالة الزر وعدد الطلبات
PATCH	/admin/trip-requests/routes/:id	تفعيل/تعطيل زر "Request a Trip" لروت معين
POST	/admin/trip-requests/notify-route/:routeId	إرسال notification يدوي لكل من طلب الروت ده
Trips Notification (تلقائي): لما الأدمن يضيف Trip على أي Route، يتبعت Notification فوراً لكل الـ passengers اللي سبق طلبوا الروت ده.
🖥️ Admin Dashboard
•	الـ Trip Requests page بقى فيها 3 Tabs: Analytics / Requests / Route Settings
•	Route Settings tab: كل الروتس بـ Switch لتفعيل/تعطيل الزر + زر "Notify" للإرسال اليدوي
•	اتشال الـ Feature Flags من Settings نهائياً

📋 API Endpoints للـ Passenger App
أعطي المطوّر ده:
BASE URL: https://your-domain.com
1. معرفة الروتس اللي فعّل فيها الزر
GET /api/trip-requests/enabled-routes
Response:
{
  "data": [
    { "id": 1, "name": "Cairo → Giza", "nameAr": "...", "fromLocation": "...", "toLocation": "..." }
  ]
}
ملاحظة: الـ GET /routes العادي كمان بقى يرجع requestsEnabled: true/false لكل روت.
2. إرسال طلب رحلة (Passenger فقط)
POST /api/trip-requests
Authorization: Bearer <passenger_token>
Body:
{
  "routeId": 1,
  "direction": "one_way" | "round_trip",
  "outboundTime": "08:00",       // 07:00–11:00
  "returnTime": "17:00"          // 15:00–19:00 (required if round_trip)
}
Response 201:
{
  "data": { ...request_object },
  "message": "تم استلام طلبك. سنُرسل لك إشعاراً حين تتوفر رحلات على هذا الخط.",
  "messageEn": "Your request has been received. You'll be notified when trips become available."
}
Errors:
- 403: { "error": "trip_requests_disabled" }  → الزر مش مفعّل للروت ده
- 400: validation errors
3. الـ Push Notification (تلقائي)
عندما يضيف الأدمن Trip على أي Route:
  → يتبعت Notification لكل passenger عمل Request على الروت ده
Notification payload:
{
  "title": "Route Available!",
  "titleAr": "الخط متاح الآن!",
  "body": "A trip is now available on a route you requested. Tap to book!",
  "bodyAr": "أصبحت رحلة متاحة على خط طلبته مسبقاً.",
  "category": "trip_available",
  "screen": "TripDetails",
  "entityId": <tripId>
}
Flow الكامل للـ Passenger App:
1.	GET /routes → شوف اللي فيه requestsEnabled: true
2.	لما يدوس على الروت → اعرض زر "Request a Trip" لو requestsEnabled: true
3.	لما يدوس الزر → Sheet/Modal بالـ time slots → POST /api/trip-requests
4.	→ رسالة تأكيد "تم استلام طلبك"
5.	→ Push notification تلقائي لما الأدمن يضيف Trip على الروت ده
