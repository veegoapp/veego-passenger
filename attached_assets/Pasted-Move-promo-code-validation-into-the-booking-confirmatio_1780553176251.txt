Move promo code validation into the booking confirmation step for ALL 4 services: Car, Bike, Delivery, and Shuttle.

--- BACKGROUND ---

The app has 4 services:

1. CAR — ride from A to B, shows price estimate before confirm
2. BIKE — same flow as Car, different vehicle type
3. DELIVERY — same flow as Car, different vehicle type
4. SHUTTLE — fixed routes with stations; user picks a line, selects pickup and dropoff station, sees total price before confirm

All 4 flows show the final price before the user confirms the booking.
That price is the orderAmount to send with promo validation.

--- WHAT TO BUILD ---

STEP 1 — Update validateCode signature (src/hooks/usePromos.ts)
- Change signature to: validateCode(code: string, orderAmount: number)
- Remove the default value of 0
- Always send { code, orderAmount } in the request body

STEP 2 — CAR, BIKE, DELIVERY flows
These 3 share the same structure (pick A to B, get price estimate).
For each of their confirmation sheets/screens:
- Add a promo code input field
- Use the existing price estimate value as orderAmount
- Call validateCode(code, priceEstimate) on apply
- If valid: show discount amount and update the displayed total
- Pass the validated promo code when calling the ride/delivery request API

STEP 3 — SHUTTLE flow
The shuttle confirmation sheet already knows pendingBooking.price.
- Add a promo code input field to the shuttle booking confirmation sheet
- Call validateCode(code, pendingBooking.price) on apply
- If valid: show discount amount and update the displayed total
- Pass the validated promo code when calling the shuttle booking API

STEP 4 — Promo input UI (same for all 4 flows)
Use this consistent pattern:
- Text input: placeholder "Enter promo code"
- "Apply" button next to it
- On success: show green text "Discount applied: -[amount]"
- On failure: show red text with the error message from the API
- If a code is applied, show an "x" to remove it and restore original price

--- DO NOT CHANGE ---
- The existing promo screen (app/promo.tsx) — leave it as is
- Any socket events, endpoint paths, or field names not mentioned here
- Any files unrelated to this feature
